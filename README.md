## 🔒 Telegram Fork (Private Development)

Мой главный и самый амбициозный проект — **независимый форк Telegram** на базе официальной библиотеки **TDLib**.

## 🚀 Как поднять свой мессенджер? (Концепция для форкеров)

Мы проектируем проект так, чтобы **любой желающий мог форкнуть этот репозиторий** и развернуть собственную сеть мессенджера за несколько минут. 

**Что потребуется от вас:**
1. Нажать кнопку **Fork** на этом репозитории.
2. Иметь собственную **мощную серверную часть** (VPS/VDS с хорошим процессором и SSD для обработки базы данных TDLib и трафика).
3. Запустить готовый Docker-компоуз (инструкция и скрипты будут добавлены в публичный релиз). Серверная часть автоматически развернет изолированную среду для вашей независимой копии мессенджера.

## 🌐 Рекомендуемые хостинги для развертывания (Hosting Providers)

### 🆓 Топ бесплатных и условно-бесплатных хостингов (Для тестов и Уровней A-B)

*   **Oracle Cloud Infrastructure (Free Tier)**
    *   *Что дают*: До 4 ядер CPU (Ampere), 24 GB RAM и 200 GB диска абсолютно бесплатно. Хватит даже для Уровня C.
*   **Hugging Face Spaces (Docker SDK) / Render / Railway**
    *   *Что дают*: Контейнеризированная среда для запуска кастомных скриптов, ботов или мелких плагинов.
*   **Vercel / GitHub Pages**
    *   *Что дают*: Пожизненный бесплатный хостинг статических страниц (промо-страница вашего форка мессенджера).

### 💰 Топ платных и абузоустойчивых хостингов (Для Highload и Уровней D-G)

*   **AnubizHost (Нидерланды / Панама)** — *Абузоустойчивый лидер*
    *   *Плюсы*: Полный **No-KYC** (без паспортов), идеален под жесткий абуз, принимает крипту (BTC, XMR).
*   **Coin.Host (Швейцария)** — *Для крипто-проектов и приватности*
    *   *Плюсы*: Серверы в Цюрихе, отличная встроенная Anti-DDoS защита на сетевом уровне. Лояльны к техническому абузу.
*   **Space-Core / Space-Hosting (Исландия / Нидерланды)**
    *   *Плюсы*: Каналы от 1 Gbps до 10 Gbps без лимитов на трафик. Игнорируют спам-жалобы и DMCA.
*   **Hetzner / OVH** — *Бюджетный Highload (Без защиты от абузов)*
    *   *Плюсы*: Самое дешевое и надежное железо в Европе. Подходит только для внутренних баз данных за прокси.

## 🗄️ Спецификация по базам данных (Database & Limits Matrix)

При развертывании форка мессенджера или софта, критически важно учитывать, как хостинг работает с базами данных (SQLite/PostgreSQL/MySQL), сколько это стоит и какие ограничения накладывает на определенные категории (A-G):

| Сервис / Провайдер | Статус Docker / DB | Доступные типы Баз Данных (DB) | Главные минусы платформы | Критично для категорий |
| :--- | :---: | :--- | :--- | :---: |
| **Oracle Cloud** | **БЕСПЛАТНО** | Встроенные в Docker (PostgreSQL, MySQL, SQLite) + Oracle NoSQL. | **Жесткий KYC.** Требует привязки зарубежной банковской карты. Любые подозрительные аккаунты банят без предупреждений. | **Все категории** (Сложно зарегистрироваться) |
| **Hugging Face** | **ПЛАТНО** | Только внешние DB (через API) или локальная SQLite **без сохранения данных** при перезапуске. | **Docker теперь платный ($9/мо)**. Диск не персистентный — при перезапуске контейнера вся встроенная БД сотрется. | **A, B, C** (Для мелких тестов) |
| **Render / Railway** | **ПЛАТНО** | PostgreSQL, Redis, MySQL (встроенные или в Docker). | Бесплатные лимиты быстро сгорают. Платные базы данных стоят дорого при росте объема записей. | **B, C, D** (Дорого при масштабе) |
| **Vercel / GitHub Pages**| **БЕСПЛАТНО** | Только внешние Serverless БД (Supabase, Neon) по API. | **Нет поддержки Docker.** Невозможно запустить серверное ядро TDLib. Только статический фронтенд. | **D, E, F, G** (Бесполезно для бэкенда) |
| **AnubizHost** | **ПЛАТНО** | Любые DB внутри вашего выделенного Docker-контейнера. | Высокая цена на аренду. Панель управления и техподдержка уступают по скорости крупным гигантам. | **A, B** (Невыгодно для малых тестов) |
| **Coin.Host** | **ПЛАТНО** | Любые DB внутри Docker. Идеально под тяжелые кластеры PostgreSQL. | **Проверка номера телефона** или email. Жесткий внутренний мониторинг за расходом ресурсов CPU. | **F, G** (Нужна точечная настройка лимитов) |
| **Space-Core** | **ПЛАТНО** | Любые DB внутри Docker на быстрых NVMe накопителях. | Периодически требуют текстовое описание проекта перед выдачей мощных серверов во избежание спама. | **E, F, G** (Критично при первом заказе) |
| **Hetzner / OVH** | **ПЛАТНО** | Идеально для огромных баз данных (RAID-массивы NVMe). | **Тотальный KYC по паспорту.** Нулевая абузоустойчивость: сервер и БД удалят при первых жалобах на спам или неофициальный API. | **D, E, F, G** (Высокий риск потерять все данные) |
